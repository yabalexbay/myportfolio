RustV OS Lab

A bare-metal RISC-V operating-system kernel written from scratch in Rust, booting on QEMU with no firmware.

RustRISC-VBare-Metal (no_std)RISC-V AssemblyQEMU
  • Preemptive multitasking: a CLINT timer-interrupt scheduler round-robins tasks with full context switching in hand-written assembly.
  • Sv39 virtual memory and user-mode syscalls over the ecall boundary; the MMU traps illegal accesses and kills only the offending task.
  • A virtio-blk driver, a custom TinyFS filesystem, and an interactive UART shell, all CI-tested by booting headless in QEMU.

02 / ABOUT

About Me

I'm a software engineer who works across the full depth of the stack, from training and serving ML models to writing the kernel-level code they ultimately run on. My experience spans real-time ML infrastructure, backend systems, and full-stack product development, including an AI recommendation engine powering a live marketplace and streaming telemetry systems built on FastAPI and WebSockets.

I like understanding systems from first principles. That's led me to build a bare-metal RISC-V operating system in Rust, a distributed database with Raft consensus and MVCC from scratch in pure Python, and a NumPy GAN with hand-written backpropagation, not because I had to, but because knowing how things work underneath makes everything I build on top better.

Good software isn't just code that works. It solves a real problem, survives failure gracefully, and leaves room for what comes next.
